Review discipline

How I prepare a contribution.

  1. Start from the repository's issue, PR template, and contribution rules.
  2. Keep the patch scoped to the reported behavior.
  3. Add focused regression coverage when the project has a practical test path.
  4. Write concrete validation notes instead of vague claims.
  5. Compare the final PR against the issue requirements before submitting.
